Exercise 1¶

We need to rank the following functions by their order of growth: $2^{\lg*(n)}, n^3, \lg*(n), e^n, n, n^2, \lg(n!), 2^n, n!, 2^{2^n}, \ln(n), (n + 1)!, n\lg(n), 1, 2^{2^{n + 1}}$.

Firstly, let's sort the functions with obvious classes of asymptotic complexity (information on that can be easily found directly in lecture slides). Namely, constant < logarithmic < linear < n-log-n < quadratic < cubic < exponential < factorial. We get:

$1 < \ln(n) < n < n\lg(n) < n^2 < n^3 < 2^n < n!$

Proving that $e^n$ grows faster than $2^n$ is quite easy (their ratio = const). Let's determine whether $(n + 1)!$ has the higher order than $n!$.

$\lim_{n \to +\infty} \frac{n!}{(n + 1)!} = \lim_{n \to +\infty} \frac{n!}{n!(n + 1)} = \lim_{n \to +\infty} \frac{1}{n + 1} = 0$.

Finally, we get that $n! = o((n + 1)!)$.

$1 < \ln(n) < n < n\lg(n) < n^2 < n^3 < 2^n < e^n < n! < (n + 1)!$

$\log^*(n)$, or iterated logarithm, is a function that iteratively applies log to an argument until it gets below one (e.g.: log(log(log(...log(n)))), the result is the number of such applications. $\log^*(n)$ grows very slowly, much slower than $\log(n)$. The number of logs is far smaller that the original argument to which all of them were applied. We can confidently (though intuitively) say that $\log^*(n) = o(\log(n))$. Moreover, we conclude that $2^{\lg^*(n)} = o(\log(n))$. At last, it's obvious that $\lg^*(n) = o(2^{\lg^*(n)})$, as in case of a simple logarithm (both these functions increase monotonically). In the end, we modify the ranking:

$1 < \lg^*(n) < 2^{\lg^*(n)} < \ln(n) < n < n\lg(n) < n^2 < n^3 < 2^n < e^n < n! < (n + 1)!$

Using Stirling's formula, we have $\lg(n!) = O(n\lg(n))$. Change the ranking respectively:

$1 < \lg^*(n) < 2^{\lg^*(n)} < \ln(n) < n < n\lg(n) = \lg(n!) < n^2 < n^3 < 2^n < e^n < n! < (n + 1)!$

There are two functions left: $2^{2^n}$ and $2^{2^{n + 1}}$. It's quite easy to prove that $2^{2^n} = o(2^{2^{n + 1}})$, so let's save some space for proving that $(n + 1)! = o(2^{2^n})$.

$\lim_{n \to +\infty} \frac{(n + 1)!}{2^{2^n}} = \lim_{n \to +\infty} \frac{2^{\log_2((n + 1)!)}}{2^{2^n}} = \lim_{n \to +\infty} \frac{2^{(n + 1)\log_2(n + 1)}}{2^{2^n}} = \lim_{n \to +\infty} 2^{(n + 1)\log_2(n + 1) - 2^n} = |n\log_2(n) = o(2^n)| = 0$.

Finally, we get the answer:

$1 < \lg^*(n) < 2^{\lg^*(n)} < \ln(n) < n < n\lg(n) = \lg(n!) < n^2 < n^3 < 2^n < e^n < n! < (n + 1)! < 2^{2^n} < 2^{2^{n + 1}}$

Or in reverse (as the task requires):

$2^{2^{n + 1}} > 2^{2^n} > (n + 1)! > n! > e^n > 2^n > n^3 > n^2 > \lg(n!) = n\lg(n) > n > \ln(n) > 2^{\lg^*(n)} > \lg^*(n) > 1$

In order to justify complexities, let's use the master method.

  1. $A(N) = A(N/2) + 1$.

$a = 1, b = 2 \implies N^{\log_21} = N^0 = 1; f(N) = 1.$
Case 2: $f(N) = \Theta(\lg^0N) = \Theta(1)$, that is, $k = 0$.
$A(N) = \Theta(\lg N) = O(\lg N)$.

  1. $B(N) = 2B(N/2) + N$.

$a = 2, b = 2 \implies N^{\log_22} = N^1 = N; f(N) = N.$
Case 2: $f(N) = \Theta(N\lg^0N) = \Theta(N)$, that is, $k = 0$.
$B(N) = \Theta(N\lg N) = O(N \lg N)$.

Let's write a program to calculate $T(N)$ for arbitraty $N$ (where $T(N)$ is either $A(N)$ or $B(N)$).

def A(N):
    if N == 1:
        return 1
    return A(N // 2) + 1


def B(N):
    if N == 1:
        return 1
    return 2 * B(N // 2) + N

def linear_search(array, target):
    """
    Finds index of target in array via linear search
    
    Returns:
        if target is in array: index of target in array, number of comparisons done
        otherwise: -1, number of comparisons done
    """
    for i in range(len(array)):
        if array[i] == target:
            return i, i + 1
        
    return -1, len(array)


def binary_search(array, target):
    """
    Finds index of target in array via binary search
    
    Returns:
        if target is in array: index of target in array, number of comparisons done
        otherwise: -1, number of comparisons done
    """
    left = 0
    right = len(array) - 1
    # auxillary variable to keep the number of comparisons
    n = 0
    
    while left <= right:
        i = left + (right - left) // 2
        n += 1
        
        if target < array[i]:
            right = i - 1
        elif target > array[i]:
            left = i + 1
        else:
            return i, n
        
    # if the target is missing in the array, return -1
    return -1, n

def do_experiment(search_func, array, samples):
        results = []
        
        for sample in samples:
            start = timer()
            res = search_func(array, sample)
            end = timer()
            elapsed = end - start
            results.append((res[0], elapsed, res[1]))
            
        return results

def benchmark(n_tests, n_words):
    linear_results = []
    binary_results = []
            
    for _ in range(n_tests):
        test_data = select_random_words(n_words - 1)
        # add some new word that is missing in list
        test_data.append('None')
        
        linear_result = do_experiment(linear_search, words, test_data)
        linear_result = sum([row[1] for row in linear_result]),\
                        sum([row[2] for row in linear_result])
        linear_results.append(linear_result)
        
        binary_result = do_experiment(binary_search, words, test_data)
        binary_result = sum([row[1] for row in binary_result]),\
                        sum([row[2] for row in binary_result])
        binary_results.append(binary_result)
        
    linear_results = mean([row[0] for row in linear_results]),\
                     mean([row[1] for row in linear_results])
    binary_results = mean([row[0] for row in binary_results]),\
                     mean([row[1] for row in binary_results])
        
    return linear_results, binary_results

def measure_search_time(search, n_tests, threshold):
    elapsed = 0
    
    for i in range(n_tests):
        # executing search of random elements with indices in [0, threshold]
        word = words[random.randint(0, threshold)]
        start = timer()
        search.execute(word)
        end = timer()
        elapsed += end - start
    
    # return mean of search time per word
    return elapsed / n_tests

Let's conduct a series of experiments:

  1. Create cache of small size, execute searches on small area of the list, measure average time to find a word; repeat the searches on a large area. Results must differ because only elements of small area were cached.
  2. Disable cache, repeat the experiments. Search times should be equal on small and large areas.
  3. Create cache of large size, execute search on large area. Results should be better than before because more elements were cached.

Conclusions¶

Cache generally speeds up running time of algorithms (we got boost in speed in up to 20 times). Nevertheless, if the cached elements are inquired too rarely, the running time decreases due to the useless calls to cache.
